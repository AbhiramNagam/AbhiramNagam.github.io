	Introduction:
		EC2 -> (Caas)Compute as a service.
	Amazon EC2 Instance Types:
		General Purpose,Compute Optimized,Memory optimized,Accelerated computing, storage optimized.
		Examples:
			General purpose: 
				It provides a balance of compute, memory, and networking resources.
				Includes small and medium databases.
			Compute optimized:
				HPC(High performance computing)
				Ideal for batch processing workloads.
			Memory optimized:This scenario might be a high-performance database or a workload that involves performing real-time processing of a large amount of unstructured data.
			Accelerated computing: floating-point number calculations, graphics processing, and data pattern matching,game streaming, application streaming.
			Storage optimized: 	
				Distributed file systems,data warehousing applications, and high-frequency online transaction processing (OLTP) systems.
				Application with high IOPS(input/output operations per second) requirements.
	
	EC2 Pricing:
		On-Demand: Ideal for short-term,irregular workloads that cannot be interrupted.

	Messaging and Queuing:
		Tightly coupled architecture: when a service B depending on service A, if serive B goes down , then service A still continous sending data to service B which will be lost for sure.
		
		Loosely coupled architecture: Single failure won't cause cascading failures. (It introduces a queue in the middle)
		considering the example in Tightly coupled architecture, eventhough service B fails, service A will continue sending data to the queue so they will reach service B after service B becomes available again.

Module 4:
	Connectivity to AWS:
		Internet gateway: Allows traffic from public to VPC
		Virtual private gateway: The data is encrypted by a VPN and it will only allow traffic from approved network to VPC.
		AWS Direct Connect: A dedicated private connection between your data center and VPC.
	Subnets and Network Access Control Lists:
		Security Group:	
			Stateful
			Only checks permission when packet is coming inside and remembers if it is allowed and doesn't check the response as it already knows if it is allowed.
			A security group is a virtual firewall that controls inbound and outbound traffic for an Amazon EC2 instance.
			By default, Security groups allow no traffic to come inside, but allows all traffic to go outside.
		Network ACL:
			Network Access Control Lists(NACLs) are used for subnets.
			A network ACL is a virtual firewall that controls inbound and outbound traffic at the subnet level.
			Stateless
			It checks everytime when a packet either tries to enter or leave the subnet.
			By default,	default NACL allows all inbound and outbound traffic, and custom  NACL denies all inbound and outbound traffic.
	Global Networking:
		Customer -->DNS Resolver-->DNS Server
		Route 53:
			Amazon Route 53 connects user requests to infrastructure running in AWS (such as Amazon EC2 instances and load balancers). It can route users to infrastructure outside of AWS.
			Another feature of Route 53 is the ability to manage the DNS records for domain names.

	Instance Stores and EBS:
		Instance Store: 
			Attatched physically to instance and data will be erased after instance stops.
		EBS:
			A block of volume not physically attatched to instance, and UPDATES ONLY THE NEW PART OF A FILE. And can create incremental snapshots.
			Solid State by default.
			Has HDD options.
			16TiB storage limit.
	S3:
		Common thing in S3 Standard, S3 standard IA,S3 instant retrieval,S3 flexible retrieval, S3 Glacier Deep Archive: 3 availability zones within the same region.
		Individual object size limit: 5TB.
		Unlimited Storage.
		99.999999999% durable.
		Regionally distributed.
		Serverless.	
		S3 glacier vault lock :
			write once and read many.(WORM) 
		S3 Lifecycle policy:
			Move data between different S3 tiers based on time.
		S3 Intelligent tiering:
			monitors objects to change tiers of S3 from standard to IA and vice versa if (based on last 30 days usage).
	
	EFS:
		EFS vs EBS:
			EBS:
				Need to be in same availability zone as the instance.(Availability zone level resource)
				Volumes do not automatically scale.
			EFS:
				Linux File system
				Regional Resource.
				Automatically scales.
				uses block storage.
				On premises servers can access EFS using Direct Connect.

		AWS will not deploy the patch for the OS., it may just notify us.
	User permissions and access:
		Roles:
			Roles are temporary permissions which can be given to : AWS resources, Users, External identies, Applications.
			Role is the highest priority. i.e, if someone already has a identity, if role is applied to them, then the rules in role will be applicable for them instead of old rules.
			Before an IAM user, application, or service can assume an IAM role, they must be granted permissions to switch to the role.
			When someone assumes an IAM role, they abandon all previous permissions that they had under a previous role and assume the permissions of the new role. 
		
		IAM(Identity and access management) provides authentication and authorization as service.
		
		Groups/users can have policies .
		Users can be assigned to groups.
		
		IAM policy:
			An IAM policy is a document that allows or denies permissions to AWS services and resources.  
			It is attached to users/groups.
	Aws organizations:
		SCPs can be applied to OUs, and individual member account.
		In AWS Organizations, you can apply service control policies (SCPs) to the organization root, an individual member account, or an OU. An SCP affects all IAM users, groups, and roles within an account, including the AWS account root user.
		You can apply IAM policies to IAM users, groups, or roles. You cannot apply an IAM policy to the AWS account root user.
